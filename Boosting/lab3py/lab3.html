# in: labels - N vector of class labels
# out: prior - C x 1 vector of class priors
def computePrior(labels, W=None):
    Npts = labels.shape[0]
    if W is None:
        W = np.ones((Npts,1))/Npts
    else:
        assert(W.shape[0] == Npts)
    classes = np.unique(labels)
    Nclasses = np.size(classes)

    prior = np.zeros((Nclasses,1))

    # Assignment 2.1
    #Class is a reserved keyword, so how is this used on pg.4 in lab description??
    for jdx, klass in enumerate(classes):
        agg = W[np.where(labels==klass)[0],:]
        prior[jdx] = (agg.sum(axis=0))

    return prior

# NOTE: you do not need to handle the W argument for this part!
# in:      X - N x d matrix of N data points
#     labels - N vector of class labels
# out:    mu - C x d matrix of class means (mu[i] - class i mean)
#      sigma - C x d x d matrix of class covariances (sigma[i] - class i sigma)
def mlParams(X, labels, W=None):
    assert(X.shape[0]==labels.shape[0])
    Npts,Ndims = np.shape(X)
    classes = np.unique(labels)
    Nclasses = np.size(classes)

    if W is None:
        W = np.ones((Npts,1))/float(Npts)

    mu = np.zeros((Nclasses,Ndims))
    sigma = np.zeros((Nclasses,Ndims,Ndims))

    # Assignment 1
    for jdx, (klass, W_) in enumerate(zip(classes, W)):
        idx = np.where(labels==klass)[0] # Extract the indices for which y==class is true,
        xlc = X[idx,:]
        wlc = W[idx,:]
        mu[jdx] = sum([wi*xi for wi,xi in zip(wlc, xlc)])/ wlc.sum(axis=0)
        
        for dim_idx in range(Ndims):
            var = 0
            for class_vect, W_value in zip(xlc, wlc):
                var += (W_value*(class_vect[dim_idx] - mu[jdx][dim_idx])**2)
            var = var / wlc.sum(axis=0)
            sigma[jdx, dim_idx, dim_idx] = var

    return mu, sigma

# in:      X - N x d matrix of M data points
#      prior - C x 1 matrix of class priors
#         mu - C x d matrix of class means (mu[i] - class i mean)
#      sigma - C x d x d matrix of class covariances (sigma[i] - class i sigma)
# out:     h - N vector of class predictions for test points
def classifyBayes(X, prior, mu, sigma):

    Npts = X.shape[0]
    Nclasses,Ndims = np.shape(mu)
    logProb = np.zeros((Nclasses, Npts))

    
    # Assignment 2.2
    for jdx in range(Nclasses):
        diff = X - mu[jdx] 
        log_sigma = - np.log(np.linalg.det(sigma[jdx])) / 2
        log_prior = np.log(prior[jdx])
        for i in range(Npts):
            logProb[jdx][i] = log_sigma - np.inner(diff[i]/np.diag(sigma[jdx]), diff[i]) / 2 + log_prior
    
    # one possible way of finding max a-posteriori once
    # you have computed the log posterior
    h = np.argmax(logProb,axis=0)
    return h

class BayesClassifier(object):
    def __init__(self):
        self.trained = False

    def trainClassifier(self, X, labels, W=None):
        rtn = BayesClassifier()
        rtn.prior = computePrior(labels, W)
        rtn.mu, rtn.sigma = mlParams(X, labels, W)
        rtn.trained = True
        return rtn

    def classify(self, X):
        return classifyBayes(X, self.prior, self.mu, self.sigma)

Assignment 3¶

(1) When can a feature independence assumption be reasonable and when not?¶

Feature independence can be reasonable where the correlation between variables is of less significance. For instance, someone's online shopping history can be used to classify them into a certain customer categories (impulse shopper, frequent buyer etc). There might be a correlation between the different purchases, but the value add might be so small that independence is a simpler way of viewing the problem and modeling it.

In contrast, for most NLP problems there will be a huge correlation between a word and the following word. If it's more of a 'classify a text as a type of factual category', word independence might work (texts about the ocean will mention the ocean, and you don't really care where it does so). But if you're trying to build an autocorrect system, not looking at the context around a word, i.e. the dependence of a word on other surrounding ones, that system is likely worse than one that takes feature dependence into consideration.

 (2) How does the decision boundary look for the Iris dataset? How could one improve the classification results for this scenario by changing classifier or, alternatively, manipulating the data?¶

The decision boundary looks good for the clearly separable Class 0 and Class 1, albeit it could be high variance seeing as purple looks to trend towards x = 0 the higher the y value. The Class 1 / Class 2 boundary is worse, and predicts samples from both classes incorrectly. It could be that Naive Bayes is a bit too naive for this problem, and that using a stronger classifier such as an SVM might be helpful. However, I can see how an SVM might overfit and thus introduce high variance to the problem too. Another trick could be to manipulate the data, for instance using feature scaling. It looks like multiplying the data or normalizing it might separate Class 1 and Class 2 more, while retaining the separation between Class 0 and Class 1.

# in: base_classifier - a classifier of the type that we will boost, e.g. BayesClassifier
#                   X - N x d matrix of N data points
#              labels - N vector of class labels
#                   T - number of boosting iterations
# out:    classifiers - (maximum) length T Python list of trained classifiers
#              alphas - (maximum) length T Python list of vote weights
def trainBoost(base_classifier, X, labels, T=10):
    # these will come in handy later on
    Npts,Ndims = np.shape(X)

    classifiers = [] # append new classifiers to this list
    alphas = [] # append the vote weight of the classifiers to this list

    # The weights for the first iteration
    wCur = np.ones((Npts,1))/float(Npts)

    for i_iter in range(0, T):
        # a new classifier can be trained like this, given the current weights
        classifiers.append(base_classifier.trainClassifier(X, labels, wCur))

        # do classification for each point
        vote = classifiers[-1].classify(X)

        # Get the total number of classes to iterate over
        classes = np.unique(labels)

        epsilon = 0
        
        for jdx in classes:
            idx = np.where(vote == jdx)[0]
            epsilon += np.sum(np.transpose(wCur[idx]) *  (1 - (jdx == labels[idx])))

        alpha = (np.log(1 - epsilon) - np.log(epsilon)) / 2
        alphas.append(alpha)

        wOld = wCur
        # Update the weights accordingly
        for i in range(Npts):
            wCur[i] = wOld[i] * np.exp(alpha * (-1)**(vote[i]==labels[i]))
        wCur = wCur / np.sum(wCur)
        
    return classifiers, alphas

# in:       X - N x d matrix of N data points
# classifiers - (maximum) length T Python list of trained classifiers as above
#      alphas - (maximum) length T Python list of vote weights
#    Nclasses - the number of different classes
# out:  yPred - N vector of class predictions for test points
def classifyBoost(X, classifiers, alphas, Nclasses):
    Npts = X.shape[0]
    Ncomps = len(classifiers)

    # if we only have one classifier, we may just classify directly
    if Ncomps == 1:
        return classifiers[0].classify(X)
    else:
        votes = np.zeros((Npts,Nclasses))

        for i in range(Ncomps):
            classified = classifiers[i].classify(X)
            for j in range(Npts):
                votes[j][classified[j]] += alphas[i]

        # one way to compute yPred after accumulating the votes
        return np.argmax(votes,axis=1)

# NOTE: no need to touch this
class BoostClassifier(object):
    def __init__(self, base_classifier, T=10):
        self.base_classifier = base_classifier
        self.T = T
        self.trained = False

    def trainClassifier(self, X, labels):
        rtn = BoostClassifier(self.base_classifier, self.T)
        rtn.nbr_classes = np.size(np.unique(labels))
        rtn.classifiers, rtn.alphas = trainBoost(self.base_classifier, X, labels, self.T)
        rtn.trained = True
        return rtn

    def classify(self, X):
        return classifyBoost(X, self.classifiers, self.alphas, self.nbr_classes)

(2) Plot the decision boundary of the boosted classifier on iris and compare it with that of the basic. What differences do you notice? Is the boundary of the boosted version more complex?¶

Bayes: The boundary between Class0 and Class1 doesn't move much, but there's a big difference in the boundary between Class1 and Class2. Besides the curve to accommodate for some of the cases on the border in the middle of the data, it also curves more to the top, seemingly following the two 'outliers' in the first quadrant. This seems to more realistically fit the data, compared to the banana shape of the single naive bayes classifier.

Tree: There is no real need for a change in the classification of Class0 vs Class1, which is reflected in the bundled guess from the weak classifiers. The line doesn't move, most likely because the different trees came to the same conclusion given the training data. However, the difference is quite extreme for the boundary between Class1 and Class2, where the simple tree tried to draw a straight line separating the classes, but the boosted tree classifier creates a jagged line that somewhat follows the "true" boundary for the two classes. The different weights of the many classifiers is what creates the jagged pattern, and if we continued for a couple of interations or added more classifiers.

Assignment 7¶

 If you had to pick a classifier, naive Bayes or a decision tree or the boosted versions of these, which one would you pick? Motivate from the following criteria:¶

Outliers¶

Non-boosted trees. Boosting can make the predictions of the "non-experts" to give too much weight to outliers, and sometimes it doesn't cancel out. Naive bayes can suffer from zero probabilites with outliers, so I would go with a tree.

 Irrelevant inputs: part of the feature space is irrelevant¶

Decision trees, since information gain is used to pick the next branching feature, the irrelevant parts of the feature space truly stays irrelevant!

Predictive power¶

If I were to base it off of this lab, I'd use boosted decision trees. However, while researching the question a lot of credit was given to Naive Bayes predictive power using the concepts of prior and evidence rather than the simpler concepts of information gain and entropy. So given the literature I'd use Naive Bayes, boosted.

Mixed types of data: binary, categorical or continuous features, etc.¶

I would pick Naive bayes, since it can better accommodate for continuous features than trees. Trees can still do it, but has to potentially create a lot of cutoff points depending on the range of the continuous variable.

 Scalability: the dimension of the data, D, is large or the number of instances, N, is large, or both.¶

I'd use Naive bayes since it scales O(np), where n is the number of observations, and p the number of features. Decision trees scale O(npd), where d is the depth. So the tree will always be slower unless d = 1, upon which it is equally fast.
